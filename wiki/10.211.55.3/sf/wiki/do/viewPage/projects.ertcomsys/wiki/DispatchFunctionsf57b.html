wiki1029: DispatchFunctions
There is a function pointer called os_swst_dispatch which points to a dispatch function. This function pointer is called when dispatching a higher priority task (or several tasks) and where the caller can be on either the kernel stack or an extended stack (e.g. it is called from all API calls that can have made a higher priority task ready to run). The pointer is always present and points to one of three functions, depending on configuration:
  • os_swst_dispatch_mix (if there are both basic tasks -- BTs -- and extended tasks -- ETs -- in the system)
  • os_swst_dispatch_et (if there are only ETs in the system)
  • os_ks_dispatch_bt (if there are only BTs in the system)

There is a further function pointer called os_ks_dispatch which points to a dispatch function. This function pointer is called when dispatching a higher priority task but where the caller can be only on the kernel stack. The pointer is always present and points to one of three functions, depending on configuration:

  • os_ks_dispatch_bt (if there are only BTs in the system)
  • os_ks_dispatch_et (if there are only ETs in the system)
  • os_ks_dispatch_mix (if there are both BTs and ETs in the system)

Note that a task control block exists for the idle task, but it does not count as a BT for the above purposes (it can never be dispatched).

Example 1: the system has mixed extended and basic tasks, and thus the dispatch function pointer assignments are:

  1. void (* const os_swst_dispatch)(void) = os_swst_dispatch_mix;
  2. void (* const os_ks_dispatch)(void) = os_ks_dispatch_mix;

Example 2: the system has basic tasks only, and thus the dispatch function pointer assignments are:

  1. void (* const os_swst_dispatch)(void) = os_ks_dispatch_bt;
  2. void (* const os_ks_dispatch)(void) = os_ks_dispatch_bt;

(Note that the '(potentially) stack switch' version of the dispatch pointer points to the 'on the kernel stack' version because in a system with no extended tasks the caller must be on the kernel stack)
