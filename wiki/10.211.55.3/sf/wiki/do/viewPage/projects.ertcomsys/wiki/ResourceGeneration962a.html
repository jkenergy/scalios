wiki1037: ResourceGeneration
A ROM array called os_resources[] is generated. Each element of the array corresponds to a resource declared in OIL (but not internal resources). Element 0 corresponds to the pre-defined resource RES_SCHEDULER. Example:
  • const struct os_rescb os_resources[] = {..., ..., ...};

For each element of the ROM array a RAM structure (os_rescb_dyn) is generated. Each structure is initialized to zero and so no initializers appear in the generated code. The configuration tool is free to generate these structures in an array or as individual structures.

In the ceil field of each ROM resource array element goes the ceiling priority of the corresponding resource.

In the dyn field of each ROM resource array element points to the corresponding element in the RAM dyn array.

Two ROM variables are created to point to the first and last elements of the ROM resource array. Example:

  • const struct os_rescb *os_first_res = &os_resources[0];
  • const struct os_rescb *os_last_res = &os_resources[22];
