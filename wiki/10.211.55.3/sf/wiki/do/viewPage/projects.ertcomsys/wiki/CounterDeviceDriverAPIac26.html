wiki1095: CounterDeviceDriverAPI (Version 1)

Introduction

Scalios is an event-driven real-time operating system, where the events are typically generated via a timer device: a source of interrupts recording the happening of things in the real world. In most cases an event is the passage of time: recorded by hardware timers that are clocked by the system clock. But in some cases the timers can count events. A particular source of timer/counter events is handled via a device driver and this is accessed through an API. This API is described here (the timer driver architecture is described elsewhere).

Driver basics

The driver is structured so that there is one common set of reentrant functions, with a standard interface, for each type of counter/timer hardware unit. This means several physical timers can run on one set of shared code, with the specific settings for each counter/timer of the same type held in different generated data. The driver functions are passed a handle to a device (a DeviceId type) and they normally cast this handle to be a pointer to the type of device that the driver knows about. Each instance of the structure will have been created by the configuration tool and populated correctly.

Driver header file

Each driver has a specific name, unique within a given target, with the driver function prototypes appearing in a header file of the name of the driver. For example, if the driver that handles 'type 1 timers' is called timer1 then the standard driver functions will have prototypes in a header file called timer1.h located in the target-specific folder of the Scalios OS installation, subdirectory drivers. For example: 

#include <drivers/timer1.h

The header file contains the prototypes for the standard five functions in a driver, named as follows: 

void os_cdevicedrv_xxx_stop(DeviceId);
void os_cdevicedrv_xxx_start(DeviceId);
void os_cdevicedrv_xxx_disable_ints(DeviceId);
void os_cdevicedrv_xxx_enable_ints(DeviceId, TickType, TickType);
TickType os_cdevicedrv_xxx_now(DeviceId);

where xxx is the name of the driver (e.g. timer1).

The header file also contains the device-specific structure definition, named as follows: 

struct os_devicecb_xxx {
     ....
};

The members of this structure are hardware-specific. An instance of this struct will appear in the generated data file osgen.c for each hardware counter (handled by this driver) referenced in the model. The Scalios counter/alarm subsystem will call the driver API with a pointer to the appropriate struct, passed as a DeviceId.

'stop' function

Example: 

void os_cdevicedrv_ctmr_stop(DeviceId dev)
{
      struct os_devicecb_timer1 *timer1 = (struct os_devicecb_timer1 *)dev;

      INTERRUPT_DISABLE(timer1->icr); /* No further interrupts since stopping Scalios */
}

The 'stop' function is called by ShutdownOS(). The dev parameter refers to the specific timer being stopped. In many cases this function will be empty because the timer will simply continue counting and do nothing else. But if any target-specific work must be carried out (e.g. putting the timer into a low-power mode) then the code for this should reside in this function. The example also shows how dev is used to obtain data on the specific device (in this case to obtain the address of the interrupt control register so that the INTERRUPT_DISABLE macro can write to the appropriate register to disable interrupts).

'start' function

Example: 

void os_cdevicedrv_timer1_start(DeviceId dev)
{
	struct os_devicecb_timer1 *timer1 = (struct os_devicecb_timer1 *)dev;
	PR1 = timer1->PR1_init;
	INTERRUPT_DISABLE(timer1->icr);
	T1CON = timer1->T1CON_init;		/* Set timer counting (and other config settings) */
	*(dev->tmr) = 0;				/* Start time at zero */
	INTERRUPT_DISMISS(timer1->icr);
	INTERRUPT_ENABLE(timer1->icr);
}

This function starts the specific timer referred to by dev. The timer should, if possible, start counting from zero (although for many devices this is not possible). It is called only by Scalios when starting (i.e. from the StartOS call).

'disable_ints' function

This function disables future interrupts from the timer referred to by dev. If there are any pending interrupts they must be cleared. This function is called by the counter and alarm subsystem of Scalios when there are no further alarm events to be handled by the OS. 

void os_cdevicedrv_timer1_disable_ints(DeviceId dev)
{
      struct os_devicecb_timer1 *timer1 = (struct os_devicecb_timer1 *)dev;
      INTERRUPT_DISABLE(timer1->icr);
}
