wiki1143: Stackswitchingscenario6 (Version 2)
Scenario 6. See doc1027 doc1028 doc1029: Task E3 activates Task B2, blocks, Task B2 sends and event to Task E3 which resumes and then terminates.

Task E3 runs from StartOS() and makes a WaitEvent() call. It returns to the dispatcher on the kernel stack (via os_block() and os_save2ks()). The dispatcher runs task B2.

Task B2 then calls SetEvent(), which makes task E3 READY again. The dispatcher is called and calls os_switch2ext(). The {os_save2ks() call left on the task E3 stack resumes execution and returns. WaitEvent() returns and task E3 resumes running application code.
