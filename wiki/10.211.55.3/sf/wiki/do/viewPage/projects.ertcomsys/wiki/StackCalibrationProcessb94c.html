wiki1065: StackCalibrationProcess
There are three points in the kernel source code where offsets are applied to the current stack pointer to give a new TOS:
  1. Hook function
  2. Task entry function (basic tasks)
  3. ISR entry function

These are applied by three macros respectively:

  • OFFSET_CURTOS_HOOK()
  • OFFSET_CURTOS_TASK()
  • OFFSET_CURTOS_ISR()

The values applied to the macros are provided by the host configuration tool. Each value in turn is a calibration value plus the requested value (defined in the OIL file or equivalent). The calibration value represents the difference between the value of the stack pointer when it is read in the macro and the stack pointer as it is before the execution of first instruction of the hook, task or ISR. These calibrations may be different (and probably are) between the versions of the OS library (since different code is compiled in different versions).

A stack calibration test is built against the four libraries and this determines the values and stores them in an array.

In addition there are further stack calibration values required:

  • The stack overheads for an extended task (i.e. the difference between the start of the allocated stack, and the value of the stack pointer before the first instruction of an extended task entry function).
  • The stack overheads used by an interrupt that runs on an extended stack before it can be switched to the kernel stack.
  • The kernel stack overheads.

These are used to calculate the implemented size of the stack allocated to extended tasks and the necessary size of the kernel stack (in most C implementations the stack is allocated all the spare RAM, but the offline tool can calculate the minimum kernel stack necessary and this can be used to verify that the spare RAM is sufficient).
